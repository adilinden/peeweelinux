I had the same problem. Seems to be down to the kernel image
being called bzImage-Full instead of what the script expects, 
i.e., bzImage.
You can extract your file system, and then manually change
the name (in_your_project_dir/mnt/boot). Then copy it 
onto your target and it should work. It did for me.
