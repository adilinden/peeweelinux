Hi,

The target build functions under 0.53 are still under development. The
following is the list of targets that will work. I tested them all using
4MB, 16MB and 64MB Compact Flash and Floppy (for the last option).

Compressed ramdisk on FAT 16 partition
Compressed ramdisk on ext2 partition
Uncompressed ramdisk, boot from FAT 16 partition
Uncompressed ramdisk, boot from ext2 partition
Read-only root fs with multiple ramdisks
----- Non-flash target devices -----
Compressed ramdisk on floppy disk w/o bootloader  

The other listed targets will simply refuse to do anything since the
script/function they call do not exist yet.

There'll be a pwlconfig maintanance release shortly. It adds a -b or
--experimental switch. Without it any menu options that are not completely
developed will be "turned off". Using the switch (-b or --experimental)
will show all menu options including broken features and non implemented
features.

The "Create New Project" prompt has been enhanced with a list of sample
projects that can be picked from. This should make it easier to get that
first project of the ground and a feel for pwlconfig.

Something that has potential for some grief is a change in the kernel
naming convention. I decided to not strip the kernel version from the
kernel package name. This will cause a package not found error for the
kernel package if an older project is used with the upcoming pwlconfig.
Simply pick a 'new' kernel and select the desired files to rectify the
problem.

TTYL,
Adi
