ITs better not to run your app as root, as you know, so you would need some way of performing root operations in order to change IP address dynamically maybe the SUDO program could help here. Or, better yet, if you could change it in a startup script as root before the autologin user takes charge. Just put a script in custom/etc/rcS.d  in order to do this. Just be wary of other network config scripts that may start before or after it. 

If you need to change the application only once at setup, then you can change it in the config files. I think I setup autobootX so that it relies on a DHCP server, but you can muck around with the config files in etc in order to change this. 

Mouse and keyboard I think can be disabled in XFree86 .. I think the mouse is somehting like "AllowMouseOpenFail" or some option like that. Maybe there is something similar for keyboard as well.
